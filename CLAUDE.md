## Project: pms-app
React + Firebase로 만든 웹앱. 팀들이 **공정현황(주간/월간 진행상황)**을 입력·보고하는 시스템.
이름: TechTeam PMS (현재 v6.9.x).

### 뼈대 (핵심은 전부 `src` 안)
| 위치 | 역할 |
|------|------|
| `src/App.js` | 대장 파일. 어느 화면 보여줄지 결정 (제일 큼, ~7100줄) |
| `src/firebase.js` | 클라우드 저장소(Firebase) 연결 |
| `src/constants.js` | 고정 설정값 |
| `src/utils.js` | 자주 쓰는 도우미 기능 |
| `src/components/` | 화면 하나하나 (아래 표) |
| `dataconnect/` | 데이터 저장 구조(설계도) |
| `node_modules`, `build`, `public` | 자동 생성/부품 — 직접 안 건드림 |
| `src/V6.7.0`, `v6.8.7`, `v6.8.9` | 옛날 백업본 |

## 작업 루틴 규칙 (매일 지킬 것)
1. **시작** — 팀장님이 "오늘 시작하자" 하면 → CLAUDE.md '진행 상황'을 읽고 *지난 작업·다음 할 일*을 먼저 브리핑.
2. **코드 추가·수정** — 추측 금지. 해당 파일·줄을 실제로 확인한 뒤 진행하고, 바꾼 뒤 빌드로 점검. **빌드 점검은 팀장님 VS Code(`npm start` 자동 리컴파일 또는 `npm run build`)에서** — Claude 샌드박스의 파일 미러는 낡을 수 있어 그쪽 빌드는 신뢰 금지(2026-06-11 확인).
3. **추천 포함** — 모든 답변에 내 추천을 섞되, *왜 더 효율적인지 근거*를 함께 제시. 귀찮음 기반 추천 금지.
4. **화면 설명** — 화면 관련 질문은 진짜 자세히 설명(비전문가 눈높이).
5. **진행 방식** — 작업·학습은 한 단계씩(완료 후 다음). 단순 단답 질문은 바로 답.
6. **마무리** — 팀장님이 "오늘 작업 종료" 하면 → ① `일일리포트/2026-MM-DD_주제.md` 상세 리포트 작성(아래 양식) ② 요약 1~2줄을 '진행 상황'에 복사 ③ 깃허브 동기화 1회.
7. **깃허브** — 현재 원격 저장소 없음(최초 설정 필요). 실제 push는 인증이 있는 VS Code 쪽에서 실행, Claude는 커밋 메시지 정리·점검 지원.
8. **수치 검증 (2026-06-11 확정)** — 시스템의 모든 수치는 아직 **미검증 초안** (누구도 공식 확인한 적 없음). 따라서 로드맵 각 단계 완료 시 **수치 테스트 시뮬레이션 필수 통과 후** 다음 단계 진행: ① 자동 시나리오 테스트(로직 검증, 예: 6/11 시운전 8케이스) ② 실기기 키인 테스트(팀장님 직접) ③ 손검산 대조(화면 숫자 vs 수동 계산, 예: 6/11 공정률 5행 검산).
