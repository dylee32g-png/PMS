## ★ 지금부터의 큰 방향 (팀장님 확정 · 최우선)
- **프로젝트 List = 시스템의 메인 뼈대/원본.** 실시간으로 관리·정리되는 진짜 자료.
- **월간보고는 List에서 자료를 받아 보여주는 '보고용 창'** → 지금은 **월간보고 작업 중단.**
- 계획: 현재 List 데이터를 **원본 엑셀로 교체** → 각 항목(컬럼)이 무슨 역할인지 정확히 파악하며 재정비.
- 기능을 다 지우는 게 아님 — 쓸 만한 기존 기능은 **응용·재사용.**

## ★ 절대 원칙 (반드시 지킬 것)
1. **엑셀 파일이 절대 우선.** 시스템 설명/항목과 엑셀이 다르면 무조건 엑셀 기준.
2. **엑셀의 항목(컬럼)을 Claude가 임의로 삭제·변경 절대 금지.** 삭제·변경은 **반드시 팀장님 확인받고 함께.**
3. 엑셀에 없는(시스템에만 있는) 항목은 Claude가 판단·제안 → 팀장님과 **하나씩** 결정해 적용.

## 삭제·업로드 절차 (코드 확인 결과)
- **별도 '전체 삭제' 불필요.** 설정 메뉴 → `엑셀 업로드` → `Firebase 확정 저장` 하면, 기존 데이터를 **자동 전량 삭제 후 엑셀로 교체**함 (`handleSaveToFirebase`, ProjectListScreen.jsx 530~550줄: 기존행 writeBatch delete → 새 행 set).
- **안전장치:** `엑셀 업로드`만으론 안 지워짐 = **pending(미리보기)** 상태. 기존 Firebase는 그대로 살아있음. `Firebase 확정 저장`을 눌러야 교체됨.
- **저장 전 반드시 백업** (기존 `_pid`·실행번호·`_statusHistory`/`_changeHistory`는 엑셀에 없어 교체 시 소실):
  - 설정 메뉴 → `엑셀 생성`(handleDownload)으로 현재 데이터 내보내기.
  - ※ '엑셀 생성'이 내부필드(_pid 등)까지 포함하는지 미확인 → 완전 백업 필요하면 별도 확인.
- **권장 순서:** ① 엑셀 업로드(pending) → ② 항목 파악 + `병합 미리보기`로 기존과 차이 확인 → ③ 백업 → ④ 팀장님 확인 후 `Firebase 확정 저장`.

## 오늘 확인한 핵심 사실 (재정비에 필요)
- **표의 '날짜' 칸 = 엑셀 원본 값 그대로 표시.** 계산·자동채움 없음. 제목에 '날짜/일자/Date/일시' 들어가면 날짜 칸 인식(`isDateCol`), `toDateInputVal`은 포맷만(YYYY-MM-DD). '무슨 날짜'인지는 **엑셀 제목칸이 정함.**
- **팝업(상세/수정) ↔ 표 동기화:** 같은 행(`_id`)·같은 저장소 공유 → **순차 수정은 동기화 됨.** 단 팝업이 **'복사본'으로 열려서**(`setDetailRow({...row})`, 265·2443줄) 팝업 열어둔 채 표 수정 시 어긋나고, 팝업 저장이 행 전체를 덮어써 표 수정이 사라질 수 있음(빈틈) → 추후 'merge 저장 + 팝업 최신연동'으로 보완.
- 데이터 소스 우선순위: `pendingData`(엑셀 업로드 미저장) > `localData`(로컬 임시) > `fbRows`(Firebase). `dataSource`로 분기. Firebase는 `onSnapshot` 실시간 구독.
- 표 표시 컬럼 = `MAIN_COL_KEYWORDS`(번호·발주처·Project·진행현황·담당자·공사계약/완료·참조 등) + '공사진행' 그룹만. 나머지 컬럼은 우클릭 → 상세/수정 팝업에서.
