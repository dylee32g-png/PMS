### 2. 표 너비 자동맞춤 ★핵심
- **글자 짤림 원인 발견**: 본문칸(td)의 `truncate`(칸 너비 초과분을 "…"로 자름) + `table-fixed`(고정폭). 둘이 겹쳐 칸을 넓혀도 글자가 짤림.
- **해결**: `table-fixed → table-auto`, `truncate` 제거, colgroup은 `minWidth`만 → 칸이 내용에 맞춰 자동으로 넓어지고 글자 안 짤림. 항목 추가·삭제돼도 자동.
- 중간에 시도한 calc/width:100% 흡수 방식은 **칸 정렬이 한 칸 밀려**(colgroup 첫 col 22 여분) 진행현황이 폭발 → 롤백. 끝의 작은 빈칸은 남기고 "글자 안 짤림" 우선.

## 결정·근거
- **table-auto + truncate 제거**: 짤림의 근본 원인이 truncate. CSS 트릭(calc) 대신 브라우저 표준 자동 너비가 확실 — "글자 안 짤림 + 내용맞춤 + 항목 자동" 전부 충족.
- **엑셀 3층 코드 개선(팀장님 선택)**: 원본 그대로 업로드 가능 → 테스트 중 매번 엑셀 손 안 대도 됨.
- **2026만(팀장님 선택)**: 구조 이질적인 SM대응·과거연도 섞임 방지.
- **전체표시 + 팝업 토글(팀장님 방향)**: 기본 다 보이고 담당자가 필요없는 항목만 끔.
