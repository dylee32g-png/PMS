- **연결 시 진행실적 이전 추가.** 근거: pid만 통일하고 장부를 안 옮기면 기존 값이 옛 장부에 남아 사라져 보임(고아). 옛/새 키가 연결 코드 안에 다 있어 안전하게 이전 가능(_migratedTo 폴백 규칙과 동일).

- **합계는 보기(2달/전체)와 무관해야 함** → 계산용 ALL_WEEKS 분리.
- **팝업을 공통 베이스로**(헤더·툴바·본문·푸터 슬롯), 진행실적 1호 적용. 확장 슬롯(항목 on/off·이전값) 예약. 기존 항목·데이터·기능은 무엇도 삭제 안 함.
