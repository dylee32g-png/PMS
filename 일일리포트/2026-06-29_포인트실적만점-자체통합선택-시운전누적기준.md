- **포인트 칸 클릭 → 실적 직접 입력**. 셀 클릭/저장(commitCellEdit)이 포인트 칸이면 `포인트실적`에만 저장 → 만점(포인트)은 안 건드림. 빈 실적은 `0 / 200`으로 표시.
- **진행실적 등록 팝업에 자체/통합 선택 버튼**(푸터 "메인표 시운전: [자체][통합]", 적용 확인창에도 선택값 표시). 선택한 시운전 포인트만 메인표 실적으로. **합(자체+통합) 제거 → 자체 아니면 통합 택1**.
- **시운전 만점(totalPt) 폴백 추가**: 기존엔 pmsData(월간데이터)만 봐서 0 → 시운전%가 0으로 빠지고 메인표엔 옛 수동값(100%·50%)이 남던 버그. `row['포인트']`(상세팝업 만점) 폴백 → 자체 50÷200=25% 등 정상.
- **시운전 메인표 %·포인트 = '그 달치(getMonthSum)' → 전체 누적(accSelfPts/accIntPts) 기준**으로 변경. 합계 200이면 200÷200=100%. 주차표·주별 키인은 그대로(직원 주마다 입력, 보고만 월1회 — 팀장님 지시).
- **공정 7개(도면입수~HMI)는 손 안 댐**: 원래부터 합계 칸(itemFinalPct=cumByKey[refWKey])=메인표 동일 계산이라 이미 일치(Chrome 확인: 20%·30%·40%·50%·50%·60%·70% 전부 메인표와 일치).

- **포인트 `실적/만점`**: 만점은 상세팝업에서 한 번 입력해 고정, 실적은 메인표 직접 입력 + 진행실적 선택 둘 다 가능(팀장님 확정).
