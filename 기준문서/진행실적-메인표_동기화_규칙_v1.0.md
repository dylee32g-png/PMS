# 진행실적 ↔ 메인표/상세팝업 동기화 규칙 v1.0

작성: 2026-06-29 / 담당: 염경록 팀장 + Claude
배경: "공사 진행" 수치가 세 화면(메인표·상세팝업·진행실적 등록)에서 서로 다른 저장소에 따로 보관돼 안 이어짐. 월간 보고에 들어갈 값이 어느 게 진짜인지 모호. → 동기화 필요.

---

## 1. 핵심 결정 (팀장님 확정 2026-06-29)

1. **원본(정본) = 진행실적 등록(주차별 W1~W5)**. 담당자가 매주 여기에 입력.
2. **메인표·상세팝업·월간보고 = 진행실적에서 자동 계산된 값을 "표시만"** 하는 창.
3. **반영 시점 = "적용하기" 버튼** (담당자 확인 후 수동 반영 → 실수 덮어쓰기 방지).
4. **메인표 값 = 진행실적 주차별 누적 최신값** (항상 같아야 함).

---

## 2. 변환 규칙 (진행실적 주차별 → 메인표 9개 칸 + 포인트)

| 메인표 칸 | 진행실적 key | 변환 방식 |
|---|---|---|
| 도면입수 | drawing | 주차별 **누적 최신값** (빈 주는 직전값 이월) → % |
| I/O Map | iomap | 누적 최신값 → % |
| 화면작성 | screen | 누적 최신값 → % |
| 기준정보 | baseinfo | 누적 최신값 → % |
| PLC | plc | 누적 최신값 → % |
| ETOS | etos | 누적 최신값 → % |
| HMI | hmi | 누적 최신값 → % |
| 자체시운전 | commissioning | (해당월 포인트 합 ÷ 만점) × 100 → % |
| 통합시운전 | intCommissioning | (해당월 포인트 합 ÷ 만점) × 100 → % |
| 포인트 | 시운전 포인트 | 메인표 표시 = **`실적/만점`** (예 `50/200`). 만점=상세팝업 '포인트'(고정·읽기). 실적=**자체 또는 통합 택1**(둘 합 ✕ — 진행실적 팝업 하단 [자체]/[통합] 버튼) 또는 메인표 칸 직접 클릭 입력. 저장 필드: `포인트실적`(실적)·`포인트소스`(self/int). — 팀장님 확정 2026-06-29(합 제거, 택1) |

> 이 규칙은 이미 시스템에 **일부 구현**돼 있음 — `ProgressModal.computeApplyData()`가 PLC·ETOS·HMI·자체/통합시운전을 월간(monthlyData)으로 변환 중. **누락 = SIMPLE 4개(도면입수·I/O Map·화면작성·기준정보)**, 그리고 도착지가 월간(monthlyData)에만 닿고 **메인표(rows 필드)엔 안 닿음**.

---

## 2-1. 진척률(%) 규칙 — 팀장님 확정 2026-06-29

```
진척률 = (도면입수% + I/O Map% + 화면작성% + 기준정보%
        + PLC% + ETOS% + HMI% + 자체시운전% + 통합시운전%) ÷ 9
```

- **9개 항목 전부 평균** (이전 5개 → 9개로 변경. 공정 입력 4개도 포함). 팀장님 결정.
- 각 % 산출: 공정·제어 7개 = 주차별 누적 최신값(이월) / 시운전 2개 = 누적 포인트 ÷ 만점 × 100.
- ⚠ **코드 변경 필요**: `ProgressModal`의 pctByWeek(615~634), overallPct(636~652) — 현재 `SECONDARY_ON(3) + 시운전(2) = 5개` → `SIMPLE(4) 포함 9개`로.
- ⚠ **메인표·월간보고의 진척률/공정률도 9개로 통일** 해야 세 화면 일치 (현재 5개 기준). 구현 시 같이 맞출 것.

## 3. 반영 흐름 (목표)

```
진행실적 등록 (주차별 입력)
   │ "적용하기" 클릭
   ▼
computeApplyData()  ← 주차 → 월 변환 + 9개 전체 계산 (SIMPLE 4개 추가)
   ▼
① 메인표 rows 필드 갱신 (도면입수~통합시운전 + 포인트)  ← 신규
② 상세팝업 = 같은 rows라 자동 반영
③ 월간보고 monthlyData 갱신  ← 기존 유지
```
