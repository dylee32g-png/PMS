# 전수검사 — 시운전 포인트(누적·전월·금월) 데이터 갈래 지도 v1.0
- 작성: 2026-06-11 / 작성자: Claude (염경록 팀장 검토)
- 목적: "시운전 금월 키인해도 누적 미합산" 버그의 원인 규명 + 계산 로직 일원화 근거 자료

## 1. 결론 요약 (한 줄)
**같은 표의 누적/전월/금월 세 칸이 서로 다른 창고에서 숫자를 꺼내 쓴다.**
금월 키인은 창고A(monthlyData)에 저장되는데, 누적 칸은 창고B(progressRecords)만 보고 계산하므로 키인값이 누적에 안 더해진다.

## 2. 데이터가 사는 곳 (창고 4곳)
| # | 창고 | 위치 | 내용 | 비고 |
|---|------|------|------|------|
| A | `monthlyData[]` | projects 문서 안 | 월별 기록 {date, currPoints, plc, etos, …} | **어미 자료(기준)** |
| B | `progressRecords` | 별도 컬렉션 (키: execNo→id 폴백) | 주간 시운전 키인 {commissioning, intCommissioning} | 진행실적 팝업이 씀 |
| C | `monthlyPoints[]` | projects 문서 안 | 레거시 월별 포인트 {date, value} | 수정 팝업이 아직 씀 |
| D | 플랫 필드/`dyn_*` | projects 문서 안 | 옛 엑셀 임포트 잔재 | 폴백용 |

## 5. 버그 원인 (확정)
1. **주 원인**: 메인 표 누적 칸(R1)은 창고B만 합산. 금월 키인(W1)은 창고A에 저장 → 누적에 미반영.
   - 같은 행에서 금월 칸(R2)은 A를 우선 보므로 금월은 보임 → "금월은 찍히는데 누적이 안 늘어남" 증상과 정확히 일치.
2. **부 원인**: 수정 팝업 키인(W2)은 창고C에 저장되는데, 누적 계산(1502)은 "A에 같은 달이 없을 때만" C를 합산. handleSubmit(2994)이 저장 시 금월 A항목을 항상 만들기 때문에 → C에 키인한 금월값이 무시됨.
3. **숨은 문제**: A의 currPoints는 자체시운전 포인트만 의미(W4가 자체만 병합). 통합시운전 포인트는 B에만 존재 → A 기준으로 통일하면 통합시운전 누적의 자리가 없음.

## 6. 일원화 방안 (권고)
- 원칙: **달마다 "A 있으면 A, 없으면 B" 한 가지 규칙**으로 누적·전월·금월을 전부 계산하는 **계산 함수 1개**(예: `getPointsTriple(p)`)를 만들고, R1~R5 전부 이 함수만 쓰게 교체.
- 효과: 어디서 키인하든(W1~W4) 표·내보내기·요약이 같은 숫자. 칸끼리 모순 불가능.
- 결정 필요(팀장님): ① currPoints에 통합시운전 포함 여부(현재 자체만) ② 시운전 정산 내보내기(R4)의 자체/통합 분리 표시 유지 방식 ③ W2(수정 팝업 리스트)를 A에 직접 쓰도록 바꿀지.
