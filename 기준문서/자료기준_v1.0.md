## 1. 상태값 (2단계 분리) — ★ 구조 확정 (2026-06-10), 세부 용어 검토 중

프로젝트 상태를 한 칸이 아니라 두 칸으로 나눈다.
근거: 기술1팀 양식이 이미 분리 운영(계약현황/작업현황), 기술2팀도 사실상 '추진중'(영업)과 '진행중'(실행)으로 구분해 쓰고 있음.

### 1-A. 계약현황 (일감이 되기까지의 영업 단계)
| 순서 | 상태 | 뜻 |
|------|------|----|
| 1 | 예상 | 비공식 정보 수준, 정식 의뢰 전 (확정 2026-06-10) |
| 2 | 검토 | 의뢰 접수, 검토 중 |
| 3 | 물량 | 물량 산출 중 |
| 4 | 투심 | 투자심의 진행 중 |
| 5 | 견적 | 견적 제출, 결과 대기 |
| 6 | 수주 | 계약 확정 → 작업현황 단계로 |
| - | 보류 | 영업 단계에서 멈춤 |
| - | 취소 | 수주 전 무산 |

### 1-B. 작업현황 (수주 후 실행 단계)
| 순서 | 상태 | 뜻 |
|------|------|----|
| 1 | 대기 | 수주됐으나 작업 전 (미작업) |
| 2 | 진행 | 작업 중 |
| - | 보류 | 실행 중 멈춤 (Hold) |
| 3 | 완료 | 작업 끝 |
| 4 | 보고완료 | 완료 보고까지 끝 — **확정**: 월간보고에 포함됐는지 판단 위해 완료와 구분 유지 |
| - | 취소/삭제 | 실행 중 무산 |

### 1-C. 기존 용어 → 새 기준 매핑 (확인 필요 항목 포함)
| 기존 용어 (출처) | 새 기준 |
|------------------|---------|
| 추진중 (웹·기술2팀) | 계약현황 단계 중 하나 (검토~견적) |
| 진행중 (웹·기술2팀) | 작업현황=진행 |
| Hold/HOLD/보류 | 보류 |
| 미작업 (웹) | 작업현황=대기 |
| 검토중 (웹) | 계약현황=검토 |
| DROP (기술1팀) | 취소 |
| 금월완료 (웹) | 상태 아님 → 완료일이 금월이면 자동 표시 |
| 신규 (웹·엑셀) | 상태 아님 → 전월 보고에 없던 프로젝트면 자동 표시 — 확정 |
| sub (웹) | 상태 아님 → 모(母)공사 밑 세부 공종 구조 표시 — 확정 |
| 예상 (웹·엑셀) | 계약현황=예상 (1단계) — 확정 |
| 이전 (웹·기술2팀) | 상태 아님 → 발주처 변경 이력으로 기록 + 배지 표시 — 확정 (2026-06-10) |
| 완료 vs 보고완료 | 구분 유지 확정 — 월간보고 포함 여부 판단 목적 |

## 2. 공통 항목 (4개 팀 필수) — ★ 초안, 검토 중

### 2-A. 사람이 입력하는 항목
| 분류 | 항목 | 비고 |
|------|------|------|
| 식별 | 실행번호, 견적번호, 프로젝트명 | |
| 식별 | 지역/공장구분 (사이트) | 파주 P9·P10, 구미, 광저우, 베트남 등 — 드롭다운 |
| 식별 | 발주처, 발주처 담당자 | 드롭다운+직접입력 |
| 사람 | 담당자(팀원), 관리자 | 드롭다운 |
| ~~의뢰~~ | ~~의뢰일, 요청자, 접수자~~ | → 팀별 선택 항목으로 이동 (2026-06-10) |
| 상태 | 계약현황, 작업현황 | 1번 기준의 2단계 상태 |
| 일정 | 시작일, 종료일(예정) | |
| 규모 | Point | |
| 실행 | 금월 시운전 실적 | 매달 입력하는 핵심 숫자 |
| 실행 | 공정률(%), 자재(%) | |
| 실행 | 작업범위 (L1/L2) | |
| 내용 | 진행내용, 업무/ISSUE | 자유 텍스트 |
| 협력 | 공사업체, 업체담당자 | |
| 기타 | 폴더링크 | 기술1팀 양식에서 채택 |
| 구조 | 모-자 공종 (sub) | 큰 공사 밑 세부 공종 |

### 2-B. 자동 계산 항목 (입력 불필요)
| 항목 | 계산 방법 |
|------|-----------|
| 번호 | 자동 부여 |
| 신규 표시 | 전월 보고에 없으면 자동 |
| 금월완료 표시 | 완료일이 금월이면 자동 |
| 누적 시운전 | 매달 실적의 합 |
| 전월 값 | 지난달 저장 데이터에서 자동 |
| 잔여 수량 | Point − 누적 |
| 증감 | 금월 − 전월 |
| 월별 추이 (그래프) | 매달 저장 데이터로 자동 생성 |

## 2-1. 완료/보고완료 전환 규칙
- 완료: 작업 끝 (담당자가 변경)
- 보고완료: 해당 프로젝트가 월간보고에 포함되어 보고된 후 (보고 시점에 일괄 전환 가능 — 검토)

## 3. 팀별 선택 항목 — 초안
| 항목 | 주 사용 팀 |
|------|------------|
| 안전관리비 (금액·제출·기안) | 기술2팀 (LGD 파주) |
| 서브원 교육일지·작업일보 제출 | 기술2팀 (LGD 파주) |
| 영업견적 제출, 공사계획서 제출, 기안, 자재 | 기술2팀 |
| 의뢰 정보 (의뢰일·요청자·접수자) | 기술1팀 |

### 설계 원칙 (확정 2026-06-10)
- **항목 추가는 최대한 간단·쉽게.** 항목 정의를 코드에 박지 않고 설정으로 관리 → 나중에 새 항목이 생겨도 코딩 없이(또는 최소 수정으로) 추가 가능하게 설계.

## 4. 이력 기록 항목 — 초안
| 기록 내용 | 예시 |
|-----------|------|
| 변경일시 | 2026-06-10 |
| 변경자 | 염경록 |
| 무엇이 | 상태(작업현황) / 발주처 / 일정 등 |
| 변경 전 → 후 | 진행 → 보류, LDE → AX |
| 사유 | 투자심의 연기 |

- 기록 대상(확정): ① 계약현황·작업현황 변경 ② 발주처 변경('이전' 대체) ③ 종료일(예정) 변경 ④ 담당자 변경
- 입력 방식: 변경 시 팝업으로 사유 한 줄 입력 (간단하게)
