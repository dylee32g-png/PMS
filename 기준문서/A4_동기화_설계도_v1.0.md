## 0. 한 문장 요약
"**한 프로젝트 = 고유 ID(pid) 하나**"를 만들고, 세 장부(List·월간보고·주간기록)가 전부 pid로 연결되게 한 뒤, 엑셀 업로드가 그 연결을 부수지 못하게 보존 병합으로 바꾼다.

## 2. pid 설계
- 형식: `P-{발급시각 36진수}-{난수 3자리}` (예: `P-sx7k2a-f4q`) — 짧고 충돌 사실상 불가능, 사람 눈에도 구분 가능.
- 발급: ① List 새 행/업로드 신규 행 ② 월간보고 새 프로젝트 ③ 기존 데이터는 일괄 발급(마이그레이션). **한 번 발급되면 불변·재사용 금지.**
- 저장 위치: List 행 문서 `_pid` 필드 + projects 문서 `pid` 필드. 같은 프로젝트면 같은 값.
- 연결 방법: 자동 매칭은 6%라 불가(실측) → 기존 "실행번호 등록" 모달을 **"프로젝트 연결" 모달로 확장** (유사도 추천 유지, 선택하면 양쪽에 같은 pid 기록, 실행번호는 표시용 유지).

## 3. 구현 단계 (Phase) — 단계마다 수치 시뮬레이션 통과 후 다음으로
### A-4a. pid 기반 + 삭제 도장 (중간 규모)
- pid 발급기 + 신규 생성 경로(양쪽)에 자동 발급 부착
- 기존 데이터 일괄 pid 발급(연결 안 된 행도 각자 발급 — 연결되면 한쪽으로 통일)
- "프로젝트 연결" 모달 (List↔월간 1:1, 사람이 확인)
- 우클릭 삭제 → '삭제 도장'(상태=삭제+일시+계정, 기본 숨김)으로 교체
- ✔ 검증: pid 유일성 자동테스트(대량 발급 충돌 0), 연결 시 양쪽 pid 일치, 삭제 도장 후 데이터·이력 보존 확인
### A-4b. 주간장부 docKey 통일 + 마이그레이션 (중간)
- ProgressModal·App.js docKey 규칙: **pid 우선** (전환기: pid ‖ 실행번호 ‖ 행ID 폴백 읽기)
- 기존 progressRecords 문서(실행번호 키·구 행ID 키)를 pid 키로 복사 병합. 같은 주차 충돌 시 자동 합치지 않고 **충돌 목록을 화면으로 보고 → 사람이 선택**
- ✔ 검증: 마이그레이션 전후 프로젝트별 주간 합계 대조표 자동 생성(전=후 100%), 충돌 건 손검산
### A-4c. 엑셀 보존 병합 업로드 (대규모 — 핵심)
- 전체 삭제→새로 쓰기(P2)를 **매칭 병합**으로 교체:
  - 매칭 1순위 (연도시트, 번호) → 2순위 (연도, Project명 정규화) → 실패 시 신규
  - 매칭됨: 기존 _id·pid·실행번호·이력 유지, 엑셀 내용만 갱신(차이는 변경이력에 자동 기록)
  - 신규: 새 _id+pid 발급 / 엑셀에 없는 기존 행: **삭제하지 않고** '엑셀에 없음' 표시
  - 확정 전 요약 화면: "갱신 N · 신규 M · 엑셀에 없음 K" 확인 후 저장
- ✔ 검증: **팀장님의 6/11 최신 엑셀로 실전 시험** — 보존(실행번호·이력·pid) 자동 체크 + 갱신/신규 수 손검산. (이 시험이 곧 팀장님이 원한 "현재 버전으로 웹 업데이트")
### A-4d. 중복 5종 어긋남 가시화 (소규모)
- pid로 연결된 쌍의 이름·담당자·발주처·날짜 비교 → 다르면 양쪽 화면에 ⚠ + 차이 보기
- 상태는 어휘가 다른 게 정상(계약현황 vs 작업현황 — 기준문서 2단계)이라 비교 대신 **병기 표시**
- 한쪽 수정 → 양쪽 자동 반영(쓰기 동기화)은 C단계로 보류 (어미 단일화와 함께)
- ✔ 검증: 어긋남 감지 건수 손검산 대조
